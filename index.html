Overview

Find a brief overview on testing objectives organized by pilot below.

Pilot 1

Objectives:

  • Test and validate that Client Business Process Flows are operational
  • Finalize application configurations
  • Test end-to-end business processes using standard TE.025's
  • Validate data mapping (for conversions and integrations)
  • Test workarounds
  • Identify training and Change Management needs

Pilot 2

Objectives:

  • All technical objects are delivered
  • Configuration is complete
  • Retest all failed tests from Pilot 1
  • Perform testing of the interfaces
  • Sign-off Conversion Validation Reports

Pilot 3

Objectives:

  • Convert and validate data
  • Perform a full end-to-end testing of all Customer Business Processes using modified TE.025's
  • Test all integrations and conversions
  • Test and approve all personalizations
  • Test and approve Fast Formulas
